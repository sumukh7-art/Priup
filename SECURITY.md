# Security Policy

## Supported Versions

Since Priup is currently in the active pre-release/planning phase, we only support security updates for the latest pre-release versions.

| Version | Supported |
| ------- | --------- |
| < 0.1.0 | Limited Support (active development branch) |
| >= 0.1.0| Active Support |

---

## Reporting a Vulnerability

We take the security of Priup seriously. If you find a security vulnerability, **please do not report it in a public GitHub issue.** Instead, follow these steps to report it responsibly:

1. **Send an Email:** Contact the maintainers privately at [Nihalrodge01@gmail.com](mailto:Nihalrodge01@gmail.com) and [sumukhnikhare97@gmail.com](mailto:sumukhnikhare97@gmail.com).
2. **Provide Details:** Include as much detail as possible, such as:
   * A description of the vulnerability.
   * Steps to reproduce, or a proof of concept (PoC) script.
   * Potential impact (e.g. data leak, code injection, server crash).
3. **Response Time:** We will acknowledge receipt of your vulnerability report within 48 hours and work on a resolution as quickly as possible.

---

## Responsible Disclosure

We ask that you follow responsible disclosure principles:
* Give us reasonable time to investigate and patch the issue before making it public.
* Do not exploit the vulnerability (e.g., extracting sensitive user data or causing service outages) beyond what is needed to verify its existence.
